🚀 Getting Started

Prerequisites

  • .NET 8 SDK
  • Git

Installation

git clone https://github.com/drearyworlds/icscombiner.git
cd icscombiner
dotnet restore

Running

dotnet run

The application will start on https://localhost:5001

💡 API Endpoints

GET /calendar.ics

Returns the aggregated ICS calendar feed

curl -H "Accept: text/calendar" https://localhost:5001/calendar.ics > calendar.ics

Response:

  • Content-Type: text/calendar; charset=utf-8
  • Status: 200 OK (if at least one calendar succeeded)
  • Status: 500 (if all calendars failed)

GET /health

Health check endpoint

curl https://localhost:5001/health

Response:

{"status":"healthy","timestamp":"2024-01-15T10:30:45.123Z"}

⚙️ Configuration

Edit appsettings.json to customize the aggregator:

{
  "CalendarAggregator": {
    "CacheMinutes": 30,
    "PastDays": 30,
    "FutureDays": 540,
    "BlockedTitles": [],
    "Sources": [
      {
        "Name": "My Calendar",
        "Url": "https://example.com/calendar.ics"
      }
    ]
  }
}

Configuration Options:

  • CacheMinutes: How long to cache the merged calendar (default: 30)
  • PastDays: Include events from this many days in the past (default: 30)
  • FutureDays: Include events up to this many days in the future (default: 540)
  • BlockedTitles: Event titles to exclude (case-insensitive)
  • Sources: Array of calendar feed URLs to aggregate

🏗️ Architecture

The solution uses a layered, testable architecture with dependency injection:

  • CalendarFetcher: Downloads calendars from remote sources concurrently
  • CalendarFilter: Removes events based on date range and title rules
  • CalendarMerger: Combines events with intelligent deduplication
  • CalendarCache: Provides in-memory caching
  • CalendarAggregatorService: Orchestrates the entire pipeline

⚡ GitHub Actions Automation
